# Руководство пользователя

## Назначение

Gas Ratio Interpreter v0.3 помогает импортировать газовые данные, привести
колонки к стандартным полям, рассчитать газовые коэффициенты, построить
палетки и получить предварительную инженерную интерпретацию.

Программа не заменяет геологическую экспертизу. Все выводы нужно проверять
по ГИС, литологии, буровому контексту, фону, СПО, наращиваниям и рециркуляции.

## Рабочий сценарий

1. Запустите приложение:

```powershell
.\run_app.ps1

# или вручную:
python -m streamlit run app/streamlit_app.py
```

2. Откройте вкладку `Инструкции и документация`, если запускаете проект впервые.
3. Загрузите файл `LAS`, `CSV`, `XLSX` или `XLSM`. Для каротажных данных
   рекомендуется начинать с `LAS`.
4. Выберите набор данных. Для Excel это лист, для LAS это набор `LAS`.
5. Проверьте первые 20 строк в превью.
6. Проверьте строку заголовков.
7. Проверьте сопоставление колонок.
8. Просмотрите предупреждения.
9. Выберите режим `Ch`.
10. Выберите интервал.
11. Проверьте Pixler, ternary и depth-графики.
12. При необходимости скачайте расчетную таблицу через `Экспорт CSV`.

## Чат поддержки

В интерфейсе есть блок `Чат поддержки`. По умолчанию включен режим
`Быстро: база знаний`: он отвечает сразу по локальной документации и проверенным
Q/A-примерам. Режим `Ollama: локальная модель` можно выбрать отдельно, если нужен
ответ именно от локальной модели и вы готовы подождать.

После выбора интервала чат получает только безопасные расчетные поля
выбранной строки. Полная таблица и сырые данные файла не передаются.

Для локальной модели можно настроить `ollama` provider в локальном AI config.
Если выбранный provider не готов, чат показывает команды подготовки
локального AI runtime. Подробности описаны в `docs/ai_usage.md`.

## Статус палеток

В боковой панели Streamlit можно выбрать `Размер интерфейса`, а также отображается версия `config/palettes.json` и
предупреждение о статусе границ. Если используется черновой конфиг,
интерпретируйте зоны как визуальную подсказку, а не как утвержденную
корпоративную методику.

Подробности настройки палеток описаны в `docs/palettes.md`.

## Логи диагностики

Если приложение показывает сообщение:

```text
Подробности записаны в logs/app.log.
```

передайте разработчику последние строки файла:

```powershell
Get-Content logs/app.log -Tail 80
```

Лог помогает понять, на каком шаге возникла ошибка: чтение файла, выбор листа,
поиск заголовка, mapping, расчет или конфигурация палеток.

## Строка заголовков

Приложение ищет строку заголовков среди первых 50 строк. Если файл содержит
служебные строки сверху, например название отчета или дату выгрузки, программа
должна найти настоящую строку с `Depth`, `C1`, `C2` и другими колонками.

Если автоопределение ошиблось, укажите номер строки вручную. Нумерация в поле
интерфейса начинается с `0`.

## Mapping колонок

После выбора строки заголовков приложение пытается сопоставить входные колонки
со стандартными полями:

```text
well, depth, depth_from, depth_to,
c1, c2, c3, ic4, nc4, ic5, nc5,
co2, h2s, rop, lithology
```

Если колонка не распознана автоматически, выберите ее вручную в блоке mapping.

## LAS-файлы

LAS-файлы читаются через секции `~Curve` и `~ASCII`. Приложение поддерживает
произвольный шаг глубины и числовые значения с точкой или запятой. Значение
`NULL`, указанное в секции `~Well`, превращается в пустое значение и дальше
показывается в предупреждениях или расчетах как `NaN`.

План отдельной вкладки `LAS-редактор` находится в `docs/las_editor_plan.md`.
В этой вкладке будут проверка глубины, изменение шага, добавление недостающих
строк, ручное заполнение, автоматические стратегии заполнения, сохранение
скважин и выгрузка выбранных данных обратно в LAS/Excel/CSV.

## Предупреждения

Предупреждения не всегда означают ошибку. Они показывают, где данные неполные
или требуют внимания:

- отсутствует компонент C1-C5;
- отсутствует глубина;
- часть значений не удалось преобразовать в число;
- формула `Ch` требует подтверждения;
- выбран резервный режим `Ch`.

Если компонент C1-C5 отсутствует, он принимается равным `0`, чтобы расчет мог
продолжиться. Это обязательно нужно учитывать при интерпретации.

## Выбор интервала

Интервалы отображаются по индексу строки и глубине. Если глубина отсутствует
или полностью нечисловая, приложение использует индекс строки как техническую
глубину и показывает предупреждение.

## Экспорт

В v0.3 доступен экспорт расчетной таблицы в CSV из интерфейса. Файл содержит
исходные сопоставленные поля, расчетные коэффициенты, классификацию и текстовое
предупреждение по интерпретации.
