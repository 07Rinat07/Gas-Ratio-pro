# Руководство пользователя

## Назначение

Gas Ratio Interpreter v0.3 помогает импортировать газовые данные, привести
колонки к стандартным полям, рассчитать газовые коэффициенты, построить
палетки и получить предварительную инженерную интерпретацию.

Программа не заменяет геологическую экспертизу. Все выводы нужно проверять
по ГИС, литологии, буровому контексту, фону, СПО, наращиваниям и рециркуляции.

## Рабочий сценарий

1. Запустите приложение:

```powershell
.\run_app.ps1

# или вручную:
python -m streamlit run app/streamlit_app.py
```

2. Откройте вкладку `Инструкции и документация`, если запускаете проект впервые.
3. Если LAS нужно проверить или изменить шаг глубины, откройте вкладку
   `LAS-редактор`, подготовьте файл и нажмите `Сохранить для расчетов`.
4. Во вкладке `Работа с данными` используйте данные из LAS-редактора или
   загрузите файл `LAS`, `CSV`, `XLSX` или `XLSM` напрямую.
5. Выберите набор данных. Для Excel это лист, для LAS это набор `LAS`.
6. Проверьте первые 20 строк в превью.
7. Проверьте строку заголовков.
8. Проверьте сопоставление колонок.
9. Просмотрите предупреждения.
10. Выберите режим `Ch`.
11. Выберите интервал.
12. Проверьте Pixler, ternary и depth-графики.
13. При необходимости скачайте расчетную таблицу через `Экспорт CSV`.

## Чат поддержки

В интерфейсе есть блок `Чат поддержки`. По умолчанию включен режим
`Быстро: база знаний`: он отвечает сразу по локальной документации и проверенным
Q/A-примерам. Режим `Ollama: локальная модель` можно выбрать отдельно, если нужен
ответ именно от локальной модели и вы готовы подождать.

После выбора интервала чат получает только безопасные расчетные поля
выбранной строки. Полная таблица и сырые данные файла не передаются.

Для локальной модели можно настроить `ollama` provider в локальном AI config.
Если выбранный provider не готов, чат показывает команды подготовки
локального AI runtime. Подробности описаны в `docs/ai_usage.md`.

## Статус палеток

В боковой панели Streamlit можно выбрать `Размер интерфейса`, а также отображается версия `config/palettes.json` и
предупреждение о статусе границ. Если используется черновой конфиг,
интерпретируйте зоны как визуальную подсказку, а не как утвержденную
корпоративную методику.

Подробности настройки палеток описаны в `docs/palettes.md`.

## Логи диагностики

Если приложение показывает сообщение:

```text
Подробности записаны в logs/app.log.
```

передайте разработчику последние строки файла:

```powershell
Get-Content logs/app.log -Tail 80
```

Лог помогает понять, на каком шаге возникла ошибка: чтение файла, выбор листа,
поиск заголовка, mapping, расчет или конфигурация палеток.

## Строка заголовков

Приложение ищет строку заголовков среди первых 50 строк. Если файл содержит
служебные строки сверху, например название отчета или дату выгрузки, программа
должна найти настоящую строку с `Depth`, `C1`, `C2` и другими колонками.

Если автоопределение ошиблось, укажите номер строки вручную. Нумерация в поле
интерфейса начинается с `0`.

## Mapping колонок

После выбора строки заголовков приложение пытается сопоставить входные колонки
со стандартными полями:

```text
well, depth, depth_from, depth_to,
c1, c2, c3, ic4, nc4, ic5, nc5,
co2, h2s, rop, lithology
```

Если колонка не распознана автоматически, выберите ее вручную в блоке mapping.

## LAS-файлы

LAS-файлы читаются через секции `~Curve` и `~ASCII`. Приложение поддерживает
произвольный шаг глубины и числовые значения с точкой или запятой. Значение
`NULL`, указанное в секции `~Well`, превращается в пустое значение и дальше
показывается в предупреждениях или расчетах как `NaN`.

Вкладка `LAS-редактор` позволяет загрузить LAS, выбрать кривую глубины,
задать целевой шаг (`0.1`, `0.2`, `0.5`, `1.0`, `1.2`, `2.0` или свой),
добавить недостающие строки по сетке и выбрать заполнение: пусто, сверху,
снизу, среднее сверху/снизу или линейная интерполяция. Если LAS идет по
убывающей глубине, редактор автоматически отсортирует строки по возрастанию
глубины и покажет предупреждение об исправлении порядка.

После проверки можно вручную поправить таблицу, нажать `Сохранить для расчетов`
и перейти во вкладку `Работа с данными`. Там появится переключатель
`Использовать данные из LAS-редактора`.

В этой же вкладке можно сохранить версию скважины локально. Приложение создает
папку `data/wells/`, manifest скважины и файлы версии в форматах `CSV`, `XLSX`
и `LAS`. Сохраненные скважины отображаются в блоке `Сохраненные скважины`: там
можно скачать нужную версию или загрузить ее обратно в текущие расчеты.

## LAS-корреляция

Вкладка `LAS-корреляция` предназначена для загрузки нескольких LAS-файлов и
сравнения скважин по глубине. Она не требует предварительного mapping под
газовые компоненты: приложение сохраняет исходные кривые LAS, определяет кривую
глубины и группирует параметры в `ГИС`, `Газы`, `Газовые коэффициенты` и
`Прочие`.

Базовый сценарий:

1. Откройте вкладку `LAS-корреляция`.
2. Загрузите один или несколько LAS-файлов.
3. Проверьте summary: строки, depth curve, диапазон глубин, количество ГИС- и
   газовых кривых.
4. Выберите скважины для вывода.
5. Выберите группы слева (`GR`, resistivity, density/neutron) и справа
   (`total gas`, `C1-C5`, gas ratios).
6. При необходимости задайте ручной интервал глубины.
7. Проверьте график: глубина идет сверху вниз по возрастанию.
8. Для печати нажмите `HTML для печати корреляции` и распечатайте файл из
   браузера.

Если корпоративные LAS используют нестандартные мнемоники, проверьте блок
`Кривые по скважинам`. Автораспознавание помогает быстро начать работу, но
ручной override кривых и масштабов запланирован следующим этапом.

Подробная дорожная карта находится в `docs/las_correlation_plan.md`.

## Интерпретационные графики

После расчета приложение сохраняет текущую расчетную таблицу для вкладки
`Интерпретационные графики`. Во вкладке можно выбрать диапазон глубин вручную
или оставить весь интервал, включить автомасштаб X-оси или задать минимальное и
максимальное значение вручную.

Глубина на графиках отображается по геологической логике: меньшая глубина сверху,
большая глубина ниже. Для печати используйте выгрузку `HTML для печати`: файл
открывается в браузере и печатается стандартной командой браузера.

При загрузке нескольких LAS-файлов во вкладке `Работа с данными` каждый файл
добавляется как отдельный набор данных. Выберите нужный набор перед mapping и
расчетом.

## Предупреждения

Предупреждения не всегда означают ошибку. Они показывают, где данные неполные
или требуют внимания:

- отсутствует компонент C1-C5;
- отсутствует глубина;
- часть значений не удалось преобразовать в число;
- формула `Ch` требует подтверждения;
- выбран резервный режим `Ch`.

Если компонент C1-C5 отсутствует, он принимается равным `0`, чтобы расчет мог
продолжиться. Это обязательно нужно учитывать при интерпретации.

## Выбор интервала

Интервалы отображаются по индексу строки и глубине. Если глубина отсутствует
или полностью нечисловая, приложение использует индекс строки как техническую
глубину и показывает предупреждение.

## Экспорт

В v0.3 доступен экспорт расчетной таблицы в CSV из интерфейса. Файл содержит
исходные сопоставленные поля, расчетные коэффициенты, классификацию и текстовое
предупреждение по интерпретации.
