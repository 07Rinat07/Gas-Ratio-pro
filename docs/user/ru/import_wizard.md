# Мастер импорта

Мастер импорта проводит файл через этапы выбора, предпросмотра, настройки, быстрой проверки качества и регистрации Dataset.

## Пакетный импорт

Можно обработать несколько файлов одной операцией. Ошибка одного файла не останавливает остальные. Для каждого элемента сохраняется отдельный статус, код ошибки и идентификатор зарегистрированного Dataset.

## Готовность данных

После metadata preview и quick QC рассчитывается показатель готовности от 0 до 100. Статусы: `ready`, `review`, `blocked`. Показатель является рекомендацией и не заменяет инженерную проверку.

## Безопасность

Исходные файлы сохраняются неизменяемыми. Состояние мастера не содержит содержимого файлов, DataFrame или объектов сторонних библиотек.

## Пакетный импорт и фоновые задачи

В Data Workspace откройте «Профессиональный мастер импорта», выберите несколько файлов и нажмите «Запустить импорт». Каждый файл обрабатывается независимо. В таблице задач отображаются статус, прогресс, число успешных и ошибочных элементов. Для завершённой задачи можно повторить только неудачные файлы. Терминальная история сохраняется в проекте в `imports/history.jsonl`.


## Восстановление, отмена и история

После перезапуска незавершённая задача получает статус `interrupted` и не запускается повторно автоматически. Пользователь может явно повторить её. Активную задачу можно отменить. Историю можно фильтровать, искать и выгружать в JSON или CSV. Команда очистки staging удаляет только файлы, которые не используются активными задачами.


## Готовность проекта и история в Project Explorer

Мастер показывает среднюю готовность зарегистрированных наборов данных и количество наборов со статусами «Готов», «Требует проверки», «Заблокирован» и «Без оценки». В Project Explorer история импортов загружается только после раскрытия ветки **Import history**.
