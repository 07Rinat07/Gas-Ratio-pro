# Visualization Performance Memory Budget v93

## Назначение

Инкремент завершает базовую защиту Performance Layer от неконтролируемого роста памяти при работе с большими LAS-наборами.

## Реализовано

`VisualizationRenderModelCache` теперь ограничивается одновременно:

- максимальным количеством Render Model;
- максимальным суммарным размером сериализованных данных в байтах.

Размер измеряется по UTF-8 JSON-представлению renderer-neutral контракта. Raw DataFrame и UI-объекты в кэш не помещаются.

## Поведение

- При превышении общего бюджета удаляются наименее недавно использованные записи.
- Render Model, который сам превышает весь бюджет, отклоняется и не вытесняет полезные записи.
- Повторная запись существующего ключа корректно пересчитывает занимаемый объём.
- Точечная инвалидация и `clear()` освобождают учтённый размер.

## Наблюдаемость

Performance profile содержит:

- `cache_bytes`;
- `cache_max_bytes`;
- `cache_hits`;
- `cache_misses`;
- `cache_evictions`;
- `cache_rejections`.

Метрики сериализуются вместе с результатом pipeline и доступны QA, UI и диагностике без внедрения вычислительной логики в Presentation Layer.

## Совместимость

Существующий API сохранён. Параметр `capacity` работает как прежде. Новый `max_bytes` имеет безопасное значение по умолчанию 8 MiB.
