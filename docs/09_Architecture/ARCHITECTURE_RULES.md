# GAS RATIO PRO — Architecture Rules v5 FINAL

## Обязательные правила

1. UI не работает напрямую с файловой системой.
2. UI не выполняет прямые записи в `st.session_state` для доменного состояния проекта.
3. Одна сущность имеет один Service и один Repository path.
4. Любая таблица или список данных должны использовать Unified Manager Framework.
5. Любой менеджер должен иметь единый Toolbar: импорт, открыть, обновить, редактировать, дублировать, удалить выбранное, очистить раздел, очистить всё, экспорт, настройки.
6. Любое удаление проходит через Service -> Repository/Storage -> Cleanup -> UI refresh.
7. Dataset Manager является частью Sprint 1, а не Sprint 2.
8. Plugin API закладывается как manifest/hook/registry foundation без исполнения стороннего кода в Sprint 1.
9. Sprint 1.5 обязателен перед Sprint 2.
10. Служебные файлы ИИ и локальные кэши не входят в архив проекта.

## Запрещено

- создавать новую таблицу без Toolbar Framework;
- добавлять отдельную кнопку удаления без общего Dialog/Manager паттерна;
- создавать параллельный Repository для уже существующей сущности;
- хранить данные проекта одновременно в нескольких несогласованных местах;
- считать Sprint завершенным без compileall, тестов и ручной интеграционной проверки.
