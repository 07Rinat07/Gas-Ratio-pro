# Sprint 0 — Quality Foundation Audit

## Цель

Зафиксировать фактическое состояние проекта перед архитектурной стабилизацией. Этот документ описывает реальные точки входа, проблемные зоны и порядок дальнейшего рефакторинга.

## 1. Фактическое состояние проекта

### 1.1 Размер и структура

- Python-файлов: 271.
- Общий объем Python-кода: около 74 263 строк.
- Главная точка входа: `app/streamlit_app.py`.
- Размер `app/streamlit_app.py`: 10 111 строк.

Вывод: главный файл приложения перегружен и выполняет слишком много ролей одновременно.

### 1.2 Основные каталоги

- `app/` — Streamlit UI и большая часть пользовательских сценариев.
- `core/` — вычисления, диагностика, логирование, управление состоянием.
- `projects/` — проектные сущности, экспорты, LAS-файлы, рабочие пространства.
- `wells/` — сохраненные скважины и версии.
- `las_editor/` — редактор LAS и связанные сервисы.
- `las_correlation/` — корреляция.
- `reports/` — отчетность.
- `docs/` — документация.
- `data/` — пользовательские данные проекта.

## 2. Критические архитектурные проблемы

### 2.1 `streamlit_app.py` перегружен

Файл одновременно содержит:

- главный UI;
- Project Manager;
- Dashboard;
- LAS Editor;
- работу с сохраненными скважинами;
- работу с экспортами;
- работу с Project Explorer;
- прямое управление `st.session_state`;
- прямые вызовы repository-функций;
- частичную очистку данных.

Риск: любое точечное изменение может не затронуть реальный пользовательский сценарий или сломать соседний блок.

### 2.2 `st.session_state` используется напрямую

Найдено около 240 прямых обращений к `st.session_state` в `app/streamlit_app.py`.

Риск: состояние приложения изменяется из разных мест, нет единой точки контроля, из-за чего старые таблицы, графики и выбранные объекты могут возвращаться после `rerun`.

### 2.3 Несколько хранилищ для близких сущностей

Фактически используются:

- `data/projects/` — проектные данные, LAS внутри проекта, экспорты, датасеты;
- `data/wells/` — глобальные сохраненные скважины и версии.

Риск: удаление проекта не гарантирует удаление глобальных скважин, а удаление скважины не всегда влияет на проектные LAS-записи.

### 2.4 Recent Projects не является управляемым модулем

Блок последних проектов строится внутри Dashboard через HTML и не имеет полноценного toolbar:

- нет очистки истории;
- нет удаления записи из истории;
- нет разделения между историей и физическим проектом.

### 2.5 Таблицы не имеют единого управляющего слоя

Разные таблицы реализованы разными способами:

- `st.dataframe`;
- `st.data_editor`;
- HTML-блоки;
- pandas preview;
- отдельные кнопки без общего стандарта.

Риск: одни таблицы можно очищать, другие нельзя; одни удаляют запись с диска, другие только из session state.

## 3. Уже существующие полезные элементы

### 3.1 Project Repository

Файл: `projects/repository.py`

Есть базовые функции:

- `ProjectRecord`;
- `list_projects()`;
- `ensure_default_project()`;
- `create_project()`;
- `delete_project()`.

Проблема: repository удаляет папку проекта, но не управляет связанными глобальными сущностями в `data/wells/`, session state и UI refresh.

### 3.2 Well Repository

Файл: `wells/repository.py`

Есть:

- `WellRecord`;
- `WellVersion`;
- `list_wells()`;
- `save_well_version()`;
- `delete_well_version()`;
- `delete_well_record()`.

Проблема: это отдельное глобальное хранилище, не полностью связанное с Project Manager.

### 3.3 Project LAS Files

Файл: `projects/las_files.py`

Есть:

- `ProjectLasFile`;
- `list_project_las_files()`;
- `save_project_las_file()`;
- `delete_project_las_file()`.

Проблема: проектные LAS-файлы и глобальные well versions существуют параллельно.

### 3.4 Session State Manager

Файл: `core/session_state_manager.py`

Есть механизм очистки transient state.

Проблема: он не стал единственной точкой управления состоянием, потому что UI продолжает напрямую работать с `st.session_state`.

## 4. Приоритеты Sprint 0

### P0 — немедленно

1. Зафиксировать единые ключи состояния приложения.
2. Вынести операции Project Manager из `streamlit_app.py` в отдельный слой.
3. Разделить Recent Projects History и физические проекты.
4. Запретить прямое удаление данных из UI.
5. Добавить единый cleanup flow: Manager → Repository → Storage → Session Cleanup → UI Refresh.

### P1 — после P0

1. Ввести общий toolbar для таблиц.
2. Ввести общий dialog для опасных операций.
3. Ввести единый Project Data Manager.
4. Начать разбиение `streamlit_app.py` на workspace-модули.

### P2 — после стабилизации

1. LAS Workspace 3.0.
2. Plot Studio.
3. Correlation Workspace.
4. Geological Modeling.
