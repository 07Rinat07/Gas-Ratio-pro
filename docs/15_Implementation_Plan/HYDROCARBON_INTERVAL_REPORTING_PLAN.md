# GAS RATIO PRO — Hydrocarbon Interval Reporting Plan

## Цель

Главная практическая цель приложения — автоматически находить и печатать интервалы, где есть признаки углеводородов, с понятной инженерной интерпретацией, графиками, маркерами и выводами.

Программа должна помогать инженеру не просто открыть LAS-файл, а пройти полный рабочий цикл:

1. загрузить LAS/CSV/Excel;
2. проверить качество данных;
3. найти интервалы с признаками углеводородов;
4. объяснить, почему эти интервалы выделены;
5. показать графики с маркерами;
6. сформировать печатный отчет;
7. сохранить результаты в проектной базе.

## Приоритет реализации

Этот блок получает высокий приоритет после базового LAS Workspace 3.0, потому что именно он формирует основную ценность GAS RATIO PRO для пользователя.

## Основные модули

### 1. Hydrocarbon Interval Detector

Назначение: автоматическое выделение интервалов по LAS-кривым, газовым отношениям, петрофизическим признакам и пользовательским cutoff.

Функции:

- поиск газовых аномалий;
- поиск интервалов по C1, C2, C3, iC4, nC4, iC5, nC5;
- расчет и анализ газовых отношений;
- выделение подозрительных интервалов;
- объединение близких интервалов;
- классификация интервалов: газ, нефть, газоконденсат, смешанный тип, неопределенно;
- сохранение interval manifest.

### 2. Interpretation Engine

Назначение: формирование нормального текстового объяснения для каждого интервала.

Интерпретация должна объяснять:

- какие кривые дали признак;
- насколько сильная аномалия;
- совпадает ли интервал с коллекторскими признаками;
- есть ли подтверждение по петрофизике;
- какие риски есть: рециркуляция, СПО, фон, выброс, шум, плохое качество LAS;
- что нужно проверить дополнительно.

Важно: интерпретация должна оставаться инженерной подсказкой, а не окончательным геологическим заключением.

### 3. Marked Graph Report

Назначение: строить графики LAS с визуальным выделением интервалов.

Графики должны поддерживать:

- depth track;
- gas track;
- petrophysics track;
- lithology/reservoir markers;
- цветные зоны интервалов;
- подписи кровли и подошвы;
- маркеры типа флюида;
- экспорт в PNG/SVG/PDF;
- подготовку к печати.

### 4. Printable Interpretation Report

Назначение: сформировать нормальный отчет, который можно распечатать и приложить к анализу.

Минимальная структура отчета:

1. титульная страница;
2. сведения о проекте и скважине;
3. краткая сводка;
4. таблица интервалов с признаками углеводородов;
5. интерпретация каждого интервала;
6. графики с маркерами;
7. петрофизическая сводка;
8. предупреждения и ограничения;
9. выводы.

Форматы:

- PDF;
- XLSX;
- DOCX;
- HTML preview;
- PNG/SVG для графиков.

### 5. Petrophysics Integration

Интервалы должны связываться с петрофизикой:

- porosity;
- water saturation;
- hydrocarbon saturation;
- shale volume;
- net reservoir;
- net pay;
- permeability proxy;
- cutoff logic;
- reservoir flag.

### 6. Modeling Integration

После выделения интервалов результаты должны быть готовы для геологического моделирования:

- zone model;
- correlation markers;
- fluid contacts;
- reservoir intervals;
- facies/property modeling input;
- volumetrics input.

### 7. Toolbar and Design System

Все ключевые действия должны быть доступны через понятный toolbar/ribbon:

- Open LAS;
- Validate;
- Detect HC Intervals;
- Interpret;
- Plot with Markers;
- Petrophysics;
- Modeling;
- Create Report;
- Export;
- Print.

Дизайн всех разделов должен быть единым: карточки, кнопки, панели, таблицы, графики, предупреждения и экспортные формы должны выглядеть как части одной профессиональной системы.

## Acceptance Criteria

Блок считается готовым, если:

1. приложение выделяет интервалы с признаками углеводородов;
2. для каждого интервала есть кровля, подошва, толщина и тип признака;
3. есть текстовая интерпретация по каждому интервалу;
4. графики показывают интервалы цветными маркерами;
5. отчет можно распечатать без ручной чистки технического мусора;
6. отчет содержит таблицу, графики и выводы;
7. результаты сохраняются в Project Database;
8. workflow идет через UI → Controller → Service → Repository → Storage;
9. тесты покрывают detector, interpretation report model и export package.

## Recommended implementation sequence

1. Hydrocarbon interval data model.
2. Interval detector service.
3. Interpretation text engine.
4. Marked graph model.
5. Printable report package.
6. UI toolbar actions.
7. PDF/XLSX/DOCX export.
8. Petrophysics integration.
9. Modeling integration.
