# Hydrocarbon Interval Rule Engine v11

## Назначение

`Hydrocarbon Interval Rule Engine` является завершающим подслоем текущего модуля `Hydrocarbon Interval Engine`. Его задача — не просто присвоить интервалу тип флюида, а сохранить объяснимую цепочку принятия инженерного решения.

Проектное правило: вывод программы не должен выглядеть как абсолютный факт. Формулировки должны оставаться инженерной предварительной интерпретацией, требующей подтверждения по ГИС, литологии, керну, испытаниям и буровому контексту.

## Что добавлено в v11

- `HydrocarbonInterpretationRule` — описание проектного правила интерпретации.
- `HydrocarbonRuleTrace` — трассировка срабатывания или пропуска правила.
- `INTERPRETATION_RULES` — реестр правил внутри движка.
- `rule_traces` — полный след проверки правил по каждому интервалу.
- `applied_rule_ids` — список фактически примененных правил.
- `interpretation_status` — практический статус интервала для отчета и UI.
- корректировка `confidence_score` через прозрачный `rule_delta`.

## Текущие правила

| Rule ID | Назначение |
|---|---|
| `HC-GAS-HIGH-001` | Высокая вероятность газонасыщенного интервала |
| `HC-OIL-HIGH-001` | Высокая вероятность нефтенасыщенного интервала |
| `HC-COND-MED-001` | Возможный газоконденсатный интервал |
| `HC-MIXED-CHECK-001` | Смешанный или переходный характер флюида |
| `HC-LOW-DATA-001` | Ограниченная числовая доказательная база |
| `HC-SINGLE-SAMPLE-001` | Интервал представлен одной глубинной точкой |

## Статусы интерпретации

| Status | Смысл |
|---|---|
| `high_confidence_preliminary` | Высокая предварительная достоверность, но требуется подтверждение независимыми данными |
| `preliminary` | Обычная предварительная интерпретация |
| `requires_review` | Требуется ручная проверка из-за качества данных, одиночной точки или неопределенности |

## Принцип вывода в отчет

Основной отчет должен показывать пользователю:

1. интервал;
2. тип флюида;
3. мощность;
4. confidence;
5. статус;
6. краткое инженерное объяснение;
7. рекомендации по проверке.

Технические детали (`rule_traces`, `quality_flags`, `confidence_factors`, NaN-диагностика, количество строк) должны уходить в экспертный режим или техническое приложение.

## Ограничения

Правила v11 являются проектной логикой Gas Ratio Pro. Они не являются самостоятельной опубликованной методикой оценки запасов или продуктивности. Правила комбинируют доказательства из зарегистрированных методик и должны проверяться на реальных эталонных наборах данных.
