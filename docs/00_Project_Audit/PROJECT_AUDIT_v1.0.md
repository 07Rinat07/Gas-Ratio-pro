# GAS RATIO PRO — PROJECT AUDIT v1.0

**Фаза:** Engineering Review & Project Redesign  
**Основание:** полный пересмотр проекта перед продолжением разработки по Roadmap v4.0  
**Статус:** утверждено как исходная точка обновления  

---

## 1. Краткий вывод аудита

Проект GAS RATIO PRO получил сильную backend-основу: LAS Platform, петрофизические расчеты, mud gas interpretation, crossplots, reservoir volumetrics, property modeling, facies modeling, geostatistics, interpolation, simulation, structural modeling и validation/audit foundation. Однако текущее состояние проекта нельзя считать готовым к продолжению разработки по старому Roadmap v3.0, потому что пользовательский опыт и frontend существенно отстают от backend-архитектуры.

Ключевая проблема: многие возможности уже существуют как модули, классы и отчеты, но пользователь не видит понятного рабочего сценария, красивых инструментов, кнопок, мастеров, интерактивных графиков, полноценной печати и экспорта результатов.

Поэтому дальнейшая разработка переводится на Roadmap v4.0, где приоритет переносится с добавления новых foundation-модулей на создание профессиональных рабочих пространств, улучшение UI/UX, нормальную печать, интерактивные графики и понятные инженерные workflows.

## 2. Основные замечания пользователя, принятые в аудит

### 2.1 LAS Editor

Проблема: возможность создания LAS-файла реализована на backend-уровне, но при пустом состоянии приложения пользователь не видит инструментов создания LAS. Если файл не загружен, интерфейс выглядит так, будто создать LAS невозможно.

Требование Roadmap v4.0:

- при входе в LAS Workspace всегда показывать стартовую панель;
- добавить крупные кнопки: `Создать LAS`, `Открыть LAS`, `Импорт CSV/Excel`, `Шаблоны`, `Последние файлы`;
- мастер создания LAS должен быть доступен без предварительной загрузки файла;
- инструменты Curve Manager, Header Editor, ASCII Editor, Validator, Calculator должны раскрываться через понятные кнопки/панели.

### 2.2 Печать и экспорт

Проблема: HTML-отчеты неудобны для просмотра и распечатки.

Требование Roadmap v4.0:

- PDF export;
- PNG export;
- SVG export;
- TIFF/JPEG export;
- Excel export для таблиц;
- DOCX report export;
- печать выбранного интервала глубин;
- печать всей скважины;
- пакетный отчет по скважине с графиками и таблицами.

### 2.3 Графики интерпретации

Проблема: графики неинформативные, слабые визуально, нет ручного управления масштабом по X/Y, нет нормального zoom/pan мышью, нет настройки глубины и интервала просмотра.

Требование Roadmap v4.0:

- ручной ввод X min / X max для каждой дорожки;
- ручной ввод Depth From / Depth To;
- Auto Scale / Manual Scale;
- zoom колесом мыши;
- pan мышью;
- box zoom;
- reset zoom;
- синхронизация глубины между дорожками;
- crosshair и tooltip;
- настройка ширины дорожек;
- сохранение layout-графиков.

### 2.4 Интервалы печати

Проблема: пользователь не нашел возможность задать начальную и конечную глубину для распечатки.

Требование Roadmap v4.0:

- Print Range Panel;
- ввод `Depth From` и `Depth To`;
- печать только интервала;
- печать нескольких интервалов;
- preview перед экспортом;
- масштаб страницы и ориентация листа.

### 2.5 Маркеры и обозначения

Проблема: на графиках почти нет понятных маркеров и обозначений флюидов/контактов/интервалов.

Требование Roadmap v4.0:

- water markers;
- oil markers;
- gas markers;
- condensate markers;
- dry gas / wet gas markers;
- OWC / GOC / GWC / FWL;
- pay zone markers;
- reservoir/non-reservoir bands;
- perforation intervals;
- core intervals;
- DST/MDT/RFT markers;
- user annotations.

### 2.6 Общий отчет по LAS/скважине

Проблема: нет понятной печати результата расчетов и интерпретации по всему LAS/скважине в Excel или общем отчете с графиками.

Требование Roadmap v4.0:

- Well Interpretation Report;
- summary по всей скважине;
- таблица интервалов: глубина, флюид, confidence, pay/net/gross;
- графики по глубине;
- Excel workbook с листами: Curves, QC, Mud Gas, Petrophysics, Intervals, Volumetrics;
- PDF/DOCX отчет с графиками и выводами.

### 2.7 Треугольная диаграмма

Проблема: треугольная диаграмма не работает, не показывает и не интерпретирует данные.

Требование Roadmap v4.0:

- перевести triangular diagram в статус `broken / needs redesign`;
- проверить входные данные;
- проверить алгоритм нормализации C1–C5 или выбранных компонентов;
- реализовать fallback-сообщения при отсутствии данных;
- добавить тестовые dataset;
- добавить интерпретацию зон на диаграмме;
- добавить экспорт диаграммы.

### 2.8 Frontend и дизайн

Проблема: дизайн страниц и вкладок слабый, интерфейс выглядит как набор технических страниц, а не как профессиональное инженерное ПО.

Требование Roadmap v4.0:

- перейти к layout: Ribbon / Sidebar / Project Explorer / Workspace / Inspector / Output;
- создать красивые кнопки и карточки действий;
- заменить перегруженные страницы на рабочие пространства;
- добавить раскрывающиеся панели инструментов;
- унифицировать все формы, таблицы, кнопки, вкладки;
- привести интерфейс к единому профессиональному стилю.

### 2.9 Геомоделирование

Проблема: backend геомоделирования есть, но пользователь не видит, где и как этим пользоваться, как считать и где смотреть результат.

Требование Roadmap v4.0:

- создать Geological Modeling Workspace UI;
- добавить мастеры: Structural Model, Facies Model, Property Model, Contacts, Volumetrics;
- добавить Project Explorer для объектов геомодели;
- добавить preview, таблицы, отчеты и визуализацию;
- связать backend-модули в понятный workflow.

## 3. Главные проблемы текущего состояния

1. Backend опережает UI.
2. Foundation-модули не всегда доступны пользователю через понятные рабочие сценарии.
3. Слишком много Markdown/HTML-отчетов, мало профессионального экспорта.
4. Plot Studio и графики интерпретации требуют полной переработки.
5. LAS Creation Wizard необходимо вынести в видимую стартовую панель.
6. Геомоделирование нужно превратить из backend-модулей в полноценное рабочее пространство.
7. Roadmap v3.0 больше не отражает реальные приоритеты.
