# Hydrocarbon Interpretation Engine — Validation Dataset v2

## Назначение

Validation Dataset v2 фиксирует эталонные инженерные сценарии для регрессионной проверки Hydrocarbon Interpretation Engine. Набор не является доказательством геологической истины; он защищает проект от случайного изменения согласованной логики интерпретации.

Главный принцип проекта:

> Каждая интерпретация должна быть понятной, объяснимой и воспроизводимой.

## Версия набора

`hie-validation-dataset/v2`

## Публичные функции

- `hydrocarbon_validation_cases()` — возвращает каталог эталонных случаев.
- `hydrocarbon_validation_case_frame(case_id)` — возвращает входной DataFrame для выбранного сценария.
- `hydrocarbon_validation_catalog_rows()` — возвращает сериализуемые строки каталога для QA/документации.
- `run_hydrocarbon_validation_suite()` — прогоняет все эталонные случаи через публичный API движка.
- `validate_hydrocarbon_interval_result(result, case)` — проверяет один результат интерпретации по одному кейсу.

## Правила использования

1. Любое изменение Rule Engine, Evidence, Context, Confidence, Recommendation или Limitation должно проходить Validation Dataset.
2. Если эталонный сценарий изменился осознанно, нужно обновить описание причины в документации.
3. Claystone-перемычки не должны исчезать и не должны автоматически склеивать продуктивные интервалы.
4. Сценарии с низким качеством данных должны давать ограничения и рекомендации, а не уверенное заключение.
5. Технические детали проверки относятся к экспертному уровню и не должны попадать на первую страницу инженерного отчета.
