# Third-Party License Policy

Status: mandatory release policy

## Allowed by default after review

- MIT;
- BSD-2-Clause and BSD-3-Clause;
- Apache-2.0;
- ISC;
- Python Software Foundation License;
- public-domain or CC0 code/data where provenance is recorded.

## Conditional

- LGPL: allowed only when the integration preserves the applicable relinking and notice obligations;
- MPL-2.0: allowed only with file-level separation and required notices;
- GPL/AGPL: requires an explicit architecture and distribution decision before adoption;
- proprietary SDKs: require a valid license, documented redistribution terms and isolation behind an adapter.

## Prohibited without an explicit legal decision

- code with no license;
- copied commercial source code;
- leaked or reverse-engineered proprietary implementation details;
- datasets without clear reuse rights;
- dependencies whose redistribution obligations cannot be met.

## Required evidence

Each adopted component must record: component name, purpose, upstream project, version/range, license identifier, source URL, review status, integration type and required notices. License files and notices must be included when redistribution requires them.
