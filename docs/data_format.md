# Формат входных данных

## Поддерживаемые файлы

- `.las`
- `.csv`
- `.xlsx`
- `.xlsm`

LAS сейчас является рекомендуемым форматом для каротажных газовых данных.
Приложение читает секции `~Curve`/`~C` и `~ASCII`/`~A`, берет имена кривых из
curve section, а значение `NULL` из `~Well` заменяет на пустое значение.
Числовые значения глубин могут использовать точку или запятую: `1.2` и `1,2`
будут приведены к одному числовому виду. Шаг глубины может быть любым; отдельный
LAS-редактор для изменения шага и заполнения пропусков описан в
`docs/las_editor_plan.md`.

CSV читается с автоматическим определением разделителя. Поддерживаемые кодировки:

```text
utf-8-sig, utf-8, cp1251, latin1
```

Excel-файлы читаются по всем листам. Пользователь выбирает лист в интерфейсе.

## Числовые параметры для планшетных графиков

Для целевого планшетного renderer любая числовая колонка из LAS, CSV или Excel
может быть выбрана как параметр графика, если она сопоставлена с глубиной или
индексом строки. Это относится не только к газовым компонентам, но и к ГИС,
resistivity, density/neutron, porosity, расчетным коэффициентам и корпоративным
мнемоникам.

Рекомендуемый формат для таких колонок:

- одна числовая серия на одну колонку;
- единицы измерения указаны в LAS curve section или в понятном названии колонки;
- значения не содержат текстовых суффиксов единиц внутри ячеек;
- глубина числовая и монотонная после подготовки в LAS-редакторе.

Нечисловые колонки могут использоваться в таблицах и описании интерпретации, но
не должны строиться как кривая без явного преобразования.

## Обязательность колонок

Фиксированный порядок колонок не требуется. Приложение использует mapping.

Желательные газовые компоненты:

```text
C1, C2, C3, iC4, nC4, iC5, nC5
```

Если часть компонентов отсутствует, приложение добавит их как `0` и покажет
предупреждение. Это позволяет не ломать расчет, но снижает надежность результата.

## Поддерживаемые алиасы

Примеры распознаваемых названий:

| Стандартное поле | Примеры входных колонок |
| --- | --- |
| `depth` | `Depth`, `DEPT`, `MD`, `Measured Depth`, `Глубина` |
| `depth_from` | `From`, `Top`, `От` |
| `depth_to` | `To`, `Base`, `До` |
| `c1` | `C1`, `CH4`, `Methane`, `Метан` |
| `c2` | `C2`, `Ethane`, `Этан` |
| `c3` | `C3`, `Propane`, `Пропан` |
| `ic4` | `iC4`, `i-C4`, `IsoButane`, `Изобутан` |
| `nc4` | `nC4`, `n-C4`, `Normal Butane`, `н-Бутан` |
| `ic5` | `iC5`, `IsoPentane`, `Изопентан` |
| `nc5` | `nC5`, `Normal Pentane`, `н-Пентан` |

Полный список алиасов находится в `mapping/curve_aliases.py`.

```text
examples/sample_gas_data.csv
examples/sample_gas_data.las
```

## Рекомендации к данным

- Не объединяйте ячейки в строке заголовков.
- Не используйте несколько строк заголовков для одной таблицы.
- Числовые колонки должны содержать числа, а не текст с единицами измерения.
- Служебные строки сверху допустимы, если строка заголовков находится в первых 50 строках.
- Пустые строки внутри таблицы удаляются только если они полностью пустые.
