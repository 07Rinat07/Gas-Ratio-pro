# LAS Merge, Append and Depth Repair Specification v4.0

## Назначение

Документ фиксирует обязательные требования к LAS Editor в Roadmap v4.0.

## 1. LAS Merge Center

Редактор должен сращивать LAS-файлы и объединять данные одной скважины.

### Возможности

- объединение по глубинным интервалам;
- объединение по кривым;
- вставка ГИС-данных из другого LAS;
- добавление wireline/LWD/mud logging curves;
- выбор master LAS;
- preview результата;
- отчет изменений;
- сохранение результата как нового LAS.

## 2. LAS Append Center

Append Center вставляет в текущий LAS кривые из другого LAS.

### Политика конфликтов

- заменить;
- пропустить;
- переименовать;
- оставить обе версии.

### Depth matching

- exact;
- nearest;
- interpolate.

## 3. Depth Repair Center

Depth Repair Center исправляет глубинную шкалу LAS.

### Главное правило по убывающей глубине

Если глубина убывает, редактор должен уметь создать исправленную копию с возрастающей глубиной.

По умолчанию применяется режим `Repair Depth Vector Only`:

- значения кривых не меняются;
- значения кривых не сортируются;
- значения кривых не интерполируются;
- значения остаются в исходных строках;
- меняется только столбец глубины и header cards `STRT`, `STOP`, `STEP`.

### Почему это важно

Некоторые LAS-файлы могут быть записаны в обратном направлении из-за экспорта из стороннего ПО. Пользователь должен иметь возможность привести depth index к стандартному виду, не рискуя потерять или перемешать измеренные значения.

## 4. Safe Copy Rule

Любая операция merge/append/repair создает новый LAS-файл. Исходный LAS не перезаписывается.

## 5. Audit Trail

Каждая операция должна создавать отчет:

- исходные файлы;
- выбранный режим;
- измененные header cards;
- измененный depth range;
- добавленные кривые;
- конфликтующие кривые;
- предупреждения;
- итоговый файл.
