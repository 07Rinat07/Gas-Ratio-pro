# LAS WORKSPACE MASTER SPECIFICATION v2.0

## Назначение

`LAS Workspace 2.0 Professional` является первым полностью закрываемым рабочим пространством Roadmap v4.0. Его цель — сделать работу с LAS-файлами понятной, безопасной и пригодной для ежедневной инженерной практики.

Workspace должен быть доступен даже тогда, когда LAS-файл еще не загружен. Пользователь всегда должен видеть действия: создать LAS, открыть LAS, импортировать CSV/Excel, выбрать шаблон, открыть демонстрационный пример.

## Project Explorer

Все действия с LAS должны быть доступны через единое дерево проекта:

```text
Проект
├── Скважины
│   ├── Well-001
│   │   ├── LAS
│   │   │   ├── main.las
│   │   │   │   ├── DEPT
│   │   │   │   ├── GR
│   │   │   │   └── GAS_SUM
│   │   ├── Интерпретации
│   │   ├── Графики
│   │   ├── Корреляции
│   │   └── Отчеты
├── Источники
├── Шаблоны
├── Документы
├── Расчеты
└── Настройки
```

Если выбрана скважина, пользователь должен видеть действия: создать LAS для скважины, добавить LAS, массовый импорт.

Если выбран LAS, пользователь должен видеть действия: открыть редактор, сравнить LAS, вставить кривые из другого LAS, срастить LAS, исправить глубины, экспортировать.

Если выбрана кривая, пользователь должен видеть действия: статистика, качество, предпросмотр, настройки отображения.

## Обязательные инструменты LAS Workspace 2.0

### A.1 Home & Action Launcher

Главный экран LAS Workspace должен показывать крупные действия независимо от наличия загруженного файла.

### A.2 Project Explorer Foundation

Единое дерево проекта должно быть доступно во всех рабочих пространствах. LAS-инструменты должны вызываться не из разрозненных страниц, а из контекста выбранной скважины, LAS-файла или кривой.

### A.3 LAS Creation Wizard

Мастер создания LAS должен поддерживать:

- создание пустого LAS;
- создание из шаблона;
- создание из CSV;
- создание из Excel;
- создание из TXT;
- создание по другому LAS;
- ручное создание.

### A.4 Header Designer

Редактирование заголовка должно быть выполнено через структурированную форму, а не только через сырой текст.

### A.5 Curve Manager

Должны поддерживаться управление мнемониками, единицами, описаниями, цветами, видимостью, качеством, статистикой и источником кривой.

### A.6 ASCII Spreadsheet

ASCII-секция должна редактироваться как табличные данные: поиск, фильтрация, сортировка, вставка, копирование, массовое редактирование, Undo/Redo.

### A.7 Merge & Append Center

Инструмент должен поддерживать:

- сращивание LAS-файлов одной скважины;
- вставку ГИС/каротажных кривых из другого LAS;
- выравнивание по глубине `exact / nearest / interpolate`;
- политику конфликтов `replace / skip / suffix`;
- предварительный просмотр;
- сохранение только в новую копию.

### A.8 Depth Repair Center

Инструмент должен диагностировать:

- возрастающую глубину;
- убывающую глубину;
- перемешанную глубину;
- дубликаты глубин;
- пропущенные глубины;
- неверный `STEP`.

Правило безопасного исправления убывающей глубины:

- данные сортируются по значению глубины;
- значения кривых остаются на тех же глубинах, к которым они относились до исправления;
- если на глубине `500 м` было `GAS_SUM = 10%`, после исправления на глубине `500 м` остается `GAS_SUM = 10%`;
- кривые не интерполируются и не пересчитываются;
- оригинальный LAS не перезаписывается;
- создается новая копия LAS;
- операция фиксируется в Operation Journal.

### A.9 Diagnostics & Validator

Должны проверяться header, curves, ASCII, depth, units, comments, NULL, encoding, sections.

### A.10 Export Center

Экспорт должен поддерживать LAS, CSV, XLSX, JSON, Parquet, PDF, PNG, SVG, TIFF.

## Operation Journal

Каждая потенциально изменяющая операция должна создавать запись в журнале:

- тип операции;
- исходный объект;
- результирующий объект;
- статус;
- дата;
- была ли создана копия;
- можно ли отменить;
- краткое описание изменений.

## Undo/Redo Foundation

Все операции Repair, Merge, Append, Header Edit, ASCII Edit, Curve Edit должны проектироваться как обратимые операции. Минимальный уровень v2.0 — хранение metadata и возможность восстановить предыдущую копию.

## Compare LAS

Должно быть возможно сравнить два LAS-файла по:

- header;
- curves;
- depth range;
- step;
- units;
- ASCII statistics;
- missing curves;
- changed curves.

## Ready Checklist

LAS Workspace 2.0 считается закрытым только если пользователь может:

1. Создать LAS без предварительной загрузки файла.
2. Открыть LAS из Project Explorer.
3. Отредактировать header.
4. Управлять кривыми.
5. Редактировать ASCII-данные.
6. Срастить несколько LAS.
7. Вставить ГИС-кривые из другого LAS.
8. Исправить убывающую глубину без изменения значений на глубинах.
9. Проверить LAS.
10. Экспортировать результат в инженерные форматы.
11. Получить журнал операций.
12. Не потерять оригинальный файл.


## A.3 LAS Creation Wizard 2.0

LAS Creation Wizard 2.0 является обязательной точкой входа для создания нового LAS-файла.

### Режимы создания

- пустой LAS;
- LAS из шаблона;
- LAS по структуре другого LAS;
- LAS по CSV;
- LAS по Excel;
- ручное создание.

### Обязательные шаги мастера

1. Выбор режима создания.
2. Заполнение Header.
3. Настройка глубины `START / STOP / STEP / NULL`.
4. Настройка кривых.
5. Предпросмотр таблицы и LAS-текста.
6. Валидация.
7. Сохранение только как нового файла.

### Безопасность

Мастер создания LAS не изменяет исходные файлы. Если используется другой LAS как источник структуры, из него берутся только метаданные и список кривых. Новый результат создается отдельным файлом и фиксируется в Operation Journal.

### UI/UX

Мастер должен быть доступен из LAS Workspace Home и из Project Explorer при выборе проекта, скважины или LAS-объекта. Пользователь должен видеть статус каждого шага, таблицу ошибок, предпросмотр глубины и список кривых до сохранения.


## A.4 Header Designer 2.0

Header Designer 2.0 — структурированный редактор заголовка LAS, который работает как безопасный metadata-only инструмент.

### Поддерживаемые секции

- `~Version`;
- `~Well`;
- `~Curve`;
- `~Parameter`.

### Обязательные поля

- `VERS`;
- `WRAP`;
- `STRT`;
- `STOP`;
- `STEP`;
- `NULL`;
- `DEPT`.

### Рекомендуемые поля скважины

- `WELL`;
- `FLD`;
- `COMP`;
- `LOC`;
- `CTRY`;
- `DATE`;
- `API`;
- `UWI`;
- `SRVC`.

### Правила безопасности

1. Header Designer не изменяет секцию `~ASCII`.
2. Header Designer не изменяет значения кривых.
3. Header Designer не исправляет глубины напрямую.
4. При сохранении создается новая копия LAS.
5. Исходный LAS остается неизменным.
6. Если `STRT > STOP`, пользователь получает предупреждение и должен использовать Depth Repair Center.

### Workflow

1. Пользователь выбирает LAS в Project Explorer или вставляет LAS-текст.
2. Система парсит header-карточки.
3. Пользователь меняет поля через форму.
4. Система строит preview.
5. Выполняется validation.
6. После подтверждения создается новый LAS с обновленным header и исходной `~ASCII` секцией.
