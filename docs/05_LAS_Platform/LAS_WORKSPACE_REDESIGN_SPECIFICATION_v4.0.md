## 5. Acceptance Criteria

1. Создать LAS можно без загруженного файла.
2. Кнопка создания видна сразу.
3. Мастер создания понятен.
4. Export не перезаписывает исходный файл.
5. Созданный LAS можно сразу открыть в Plot Studio.

## A.1 Implemented UI Contract

LAS Workspace 2.0 обязан показывать стартовые действия до загрузки файла. Минимальный контракт интерфейса:

1. Заголовок `LAS Workspace 2.0`.
2. Карточка `Создать LAS` доступна всегда.
3. Мастер создания LAS принимает:
   - название скважины;
   - company/field/UWI/API;
   - START/STOP/STEP/NULL;
   - единицы глубины;
   - шаблон кривых;
   - дополнительные кривые в формате `MNEM,UNIT,DESCRIPTION`.
4. После предпросмотра созданный LAS можно:
   - посмотреть как таблицу;
   - сохранить в рабочую сессию;
   - скачать как `.las`.

Этот контракт закрывает критическое замечание: инструменты создания LAS не должны быть скрыты до загрузки существующего файла.

# LAS Workspace v4.0 Addendum — Merge, Append and Depth Repair

## 1. LAS Merge & Append Center

LAS Workspace 2.0 должен включать отдельный инструмент для объединения LAS-файлов.

### Основные сценарии

1. Срастить два LAS-файла одной скважины по глубине.
2. Вставить в текущий LAS кривые из другого LAS.
3. Добавить ГИС-данные из отдельного LAS в основной LAS.
4. Объединить mud logging LAS и wireline/LWD/GIS LAS.
5. Объединить неполные LAS-файлы, полученные из разных источников.

### Политики объединения

- `append_by_depth` — добавить интервалы по глубине;
- `merge_curves` — добавить новые кривые к существующему depth index;
- `replace_curve` — заменить выбранные кривые;
- `suffix_conflict` — переименовать конфликтующие мнемоники;
- `skip_conflict` — пропустить конфликтующие кривые;
- `interpolate_to_master_depth` — интерполировать данные второго LAS на depth grid первого LAS.

### Безопасность

- исходные LAS не изменяются;
- результат сохраняется как новый LAS;
- перед сохранением показывается preview;
- создается merge report.

## 2. LAS Depth Repair & Direction Correction

Редактор обязан исправлять LAS-файлы, где глубина записана в обратном направлении.

### Главное правило

Если глубина убывает, исправление выполняется путем создания нового возрастающего depth vector. Значения кривых остаются в исходном порядке строк и не должны перемещаться между строками, если пользователь явно не выбрал режим сортировки строк.

### Пример

Исходный LAS:

```text
DEPT   GR   RT
3000   80   12
2999   82   13
2998   78   11
```

Режим `Repair Depth Vector Only`:

```text
DEPT   GR   RT
2998   80   12
2999   82   13
3000   78   11
```

Значения `GR` и `RT` остались в тех же строках измерений; исправлен только depth vector и header cards.

### Альтернативный режим

Режим `Sort Rows With Curve Values` может быть добавлен отдельно, но должен быть явно выбран пользователем. Он сортирует строки вместе со значениями кривых и применяется только тогда, когда пользователь хочет сохранить исходную связку глубина-значения.

### Обязательные проверки

- направление глубины;
- регулярность шага;
- `STRT / STOP / STEP`;
- дубликаты;
- пропуски;
- отрицательный или несогласованный `STEP`;
- соответствие количества ASCII-строк и depth range.
