# Конфигурация палеток

Палетки настраиваются через файл:

```text
config/palettes.json
```

Цель этого файла — менять визуальные зоны Pixler и ternary без правки Python-кода.

## Важное ограничение

Границы в текущем `config/palettes.json` являются черновыми визуальными зонами.
Их нельзя считать подтвержденной корпоративной методикой. Перед production-
использованием нужно заменить значения на утвержденные линии/области.

## Структура файла

```json
{
  "version": "0.4-draft",
  "notice": "Draft visual boundaries...",
  "pixler": {
    "zones": []
  },
  "ternary": {
    "regions": []
  }
}
```

`version` отображается в боковой панели Streamlit. `notice` также выводится в
интерфейсе, чтобы пользователь видел статус методики.

## Pixler zones

Каждая зона задается так:

```json
{
  "name": "Oil",
  "y_min": 1.0,
  "y_max": 20.0,
  "color": "rgba(45, 140, 85, 0.16)"
}
```

Правила:

- `name` не должен быть пустым;
- `y_min` и `y_max` должны быть положительными числами;
- `y_min` должен быть меньше `y_max`;
- `color` задается CSS-цветом, обычно `rgba(...)`.

## Ternary regions

Каждая область задается массивами координат `a`, `b`, `c`:

```json
{
  "name": "Draft gas-prone area",
  "a": [0.55, 0.9, 0.6],
  "b": [0.25, 0.05, 0.1],
  "c": [0.2, 0.05, 0.3],
  "color": "rgba(30, 115, 190, 0.12)"
}
```

Правила:

- `a`, `b`, `c` должны иметь одинаковое количество точек;
- каждая область должна содержать минимум 3 точки;
- координаты должны быть числами;
- точки задаются в системе Plotly ternary: `a = C2/ΣC`, `b = C3/ΣC`,
  `c = nC4/ΣC`.

## Что делать после изменения конфига

Запустите тесты:

```powershell
python -m pytest
```

Затем вручную проверьте интерфейс:

```powershell
streamlit run app/streamlit_app.py
```

Загрузите `examples/sample_gas_data.csv` и убедитесь, что Pixler/ternary
отображаются без ошибок.
