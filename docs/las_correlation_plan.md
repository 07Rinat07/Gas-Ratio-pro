# План multi-LAS корреляции и ГИС/газовых графиков

Цель: добавить в приложение отдельный workflow, где пользователь может загрузить
несколько LAS-файлов, сопоставить скважины по глубине и вывести рядом ГИС-кривые
и газовые данные. Экран должен помогать быстро увидеть коллекторы, газовые пики,
флюидный характер и зоны, требующие проверки по ГИС, литологии и тестам.

## Статус

Подготовительная база уже была создана ранее, а первый рабочий слой
multi-LAS корреляции реализован сейчас:

- LAS importer читает секции `~Curve`/`~ASCII` и поддерживает `NULL`;
- рабочая вкладка уже принимает несколько файлов и дает выбор набора данных;
- расчетный pipeline умеет строить gas ratios и depth-графики;
- в `docs/mud_gas_analysis_literature.md` зафиксированы требования к трекам:
  GR, total gas, `C1-C5`, `Wh/Bh/Ch`, Pixler ratios, inverse oil indicator,
  resistivity, density/neutron и интерпретационные маркеры;
- модуль `las_correlation` готовит несколько LAS без потери исходных ГИС-кривых;
- Streamlit-вкладка `LAS-корреляция` строит соседние треки `ГИС` и `Газы` по
  глубине сверху-вниз и выгружает HTML для печати;
- добавлено ручное назначение групп кривых по каждой скважине для нестандартных
  мнемоник LAS;
- добавлен ручной X-scale для левой группы ГИС и правой группы газовых треков;
- добавлено сохранение настроек корреляции в локальный проект и применение настроек из проекта или текущей сессии;
- добавлен выбор активного проекта и создание нового проекта из вкладки `LAS-корреляция`;
- добавлена таблица выбранного интервала с CSV-экспортом;
- добавлен режим `По кривой` для сравнения одной выбранной LAS-кривой между несколькими скважинами.

Режим сравнения одной кривой реализован. Следующий шаг: улучшение HTML-печати и подготовка основы для `PNG/PDF/SVG` экспорта.

## Пользовательский сценарий

1. Пользователь открывает вкладку `LAS-корреляция`.
2. Выбирает активный проект или создает новый.
3. Загружает несколько LAS-файлов.
4. Приложение для каждого файла:
   - читает кривые;
   - определяет depth curve;
   - показывает диапазон глубин, шаг, количество строк и список кривых;
   - предлагает распознанные группы кривых: `ГИС`, `Газы`, `Газовые коэффициенты`,
     `Прочие`.
5. При необходимости пользователь включает `Ручное назначение групп кривых` и
   исправляет группу для каждой кривой.
6. Пользователь выбирает:
   - какие скважины показывать;
   - общий или индивидуальный диапазон глубин;
   - какие ГИС-кривые выводить слева;
   - какие газовые кривые/коэффициенты выводить справа;
   - ручной или автоматический масштаб X для каждого трека;
   - представление `По скважинам` или `По кривой`.
7. Пользователь сохраняет текущие настройки корреляции в проект и может применить
   их повторно после перезапуска приложения или смены экрана.
8. Приложение строит глубинные графики сверху-вниз:
   - по строкам или панелям скважин;
   - с одинаковым диапазоном глубин для корреляции;
   - с соседними колонками `ГИС` и `Газы`;
   - с таблицей выбранного интервала и CSV-экспортом.
9. Пользователь выгружает выбранный интервал или весь график в `HTML` для
   печати, позже в `PDF/PNG/SVG`.

## Кривые и распознавание

Нужен слой классификации LAS-кривых, который не удаляет незнакомые поля.

Минимальные группы:

- `depth`: `DEPT`, `DEPTH`, `MD`, `TVD`, `TVDSS`;
- `gamma`: `GR`, `GK`, `GAMMA`, `GRC`, `SGR`;
- `total_gas`: `TGAS`, `TOTALGAS`, `TG`, `GAS`, `TOTAL_GAS`;
- `gas_components`: `C1`, `C2`, `C3`, `IC4`, `NC4`, `IC5`, `NC5`, `CH4`;
- `gas_ratios`: расчетные `Wh`, `Bh`, `Ch`, `C1/C2`, `C1/C3`, `C1/C4`, `C1/C5`,
  inverse oil indicator;
- `resistivity`: `RT`, `RD`, `RDEEP`, `RSHAL`, `RS`, `P22H`, `A40H`;
- `density_neutron`: `RHOB`, `RHOZ`, `NPHI`, `TNPH`, `DPHI`;
- `drilling`: `ROP`, `WOB`, `RPM`, `FLOW`;
- `lithology`: `LITH`, `LITHOLOGY`.

Пользователь может вручную назначить группу каждой кривой, потому что реальные
LAS часто используют корпоративные мнемоники.

## Графики

Глубина всегда отображается как в каротаже: меньшая глубина сверху, большая ниже.

Минимальный набор треков:

1. `GR / lithology` - коллектор и глинистость.
2. `Resistivity` - shallow/deep сравнение.
3. `Density / neutron` - газовый эффект и пористость, если кривые есть.
4. `Total gas`.
5. `C1-C5`.
6. `Wh/Bh/Ch`.
7. `Pixler ratios`.
8. `Interpretation markers`.

Для multi-well режима нужны два представления:

- `По скважинам`: каждая скважина отдельной вертикальной группой треков.
- `По кривой`: одна выбранная кривая сравнивается между несколькими скважинами.

## Данные и хранение

Нельзя терять исходный LAS. Для каждой скважины нужно хранить:

- исходный LAS;
- подготовленный LAS после правки глубины;
- выбранные кривые и ручной mapping;
- рассчитанные газовые коэффициенты;
- настройки графиков;
- журнал предупреждений и правок;
- версии экспортов.

На первом этапе можно хранить это локально рядом с `data/wells/`, позже выделить
структуру проекта:

```text
data/projects/<project_id>/
  project.json
  correlation_settings.json
  wells/<well_id>/
    source.las
    prepared.csv
    mapping.json
    charts.json
```

## Риски и ограничения

- LAS-файлы из разных источников могут иметь разные units, datum и тип глубины
  (`MD`, `TVD`, `TVDSS`). Для настоящей корреляции нужно явно показывать, какая
  глубина используется.
- Автоматическое распознавание ГИС-кривых должно быть помощником, а не
  окончательным решением: пользователь должен иметь ручной override.
- Газовые интерпретации являются предварительными и требуют проверки по ГИС,
  литологии, буровому контексту, фону, СПО, наращиваниям и рециркуляции.
- Для больших LAS нужна оптимизация: downsampling для графиков и ограничение
  отображаемого интервала.
