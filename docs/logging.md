# Логирование

Приложение пишет диагностические события в локальный файл:

```text
logs/app.log
```

Файл создается автоматически при запуске Streamlit. Логи ротируются: когда
файл становится слишком большим, создаются резервные копии.

## Что логируется

- старт Streamlit-приложения;
- факт загрузки файла: расширение и размер;
- успешное или ошибочное чтение файла;
- выбор листа;
- найденная строка заголовков и score автоопределения;
- результат auto/manual mapping;
- количество расчетных строк и предупреждений;
- выбранный интервал;
- stack trace внутренних ошибок чтения и конфигурации.

## Что не логируется

Чтобы не утекали рабочие данные, в лог не пишутся:

- значения строк таблицы;
- полные исходные данные;
- расчетная таблица;
- пользовательские секреты;
- содержимое загруженных файлов.

## Как использовать лог

Если приложение показывает сообщение:

```text
Подробности записаны в logs/app.log.
```

Откройте файл:

```powershell
Get-Content logs/app.log -Tail 80
```

Для диагностики обычно нужны последние 50-100 строк.

## Для разработчиков

Логгер находится в:

```text
core/logging_config.py
```

Правила:

- не писать в лог сырые данные скважин и таблиц;
- не показывать stack trace пользователю в UI;
- подробности ошибок писать в `logs/app.log`;
- новые важные workflow-события покрывать тестами.

Проверка:

```powershell
python -m pytest tests/test_logging_config.py
```
