# GAS RATIO PRO — Reference Sources Specification Draft

## 1. Назначение

Reference Sources Manager предназначен для хранения, регистрации и проверки PDF-источников, которые используются в проекте как доказательная база для требований, алгоритмов, инженерных решений и документации.

## 2. Проблема

Нельзя оставлять в документации ссылки вида:

```text
<локальный абсолютный путь пользователя к PDF>
```

Такая ссылка работает только на компьютере автора и не является воспроизводимой. Любой источник, на который ссылается проект, должен быть вложен в проектный архив или в папку документации.

## 3. Правильный подход

Источник должен храниться по относительному пути:

```text
docs/sources/<source-name>.pdf
```

Рядом должен быть реестр:

```text
docs/sources/source_registry.json
```

В реестре фиксируются:

- идентификатор источника;
- название;
- авторы;
- год;
- относительный путь;
- исходное имя файла;
- назначение в проекте;
- размер;
- SHA256-контрольная сумма.

## 4. Поддерживаемые форматы

На текущем этапе поддерживается PDF. В дальнейшем можно добавить DOCX, HTML, ZIP-архивы данных и DOI-ссылки, но только после отдельной спецификации.

## 5. Минимальные функции

Reference Sources Manager должен поддерживать:

- добавление PDF-источника в проект;
- сжатие PDF без потери читаемости;
- регистрацию источника в JSON-реестре;
- проверку наличия файла;
- проверку, что в реестре не используется локальный Windows-путь;
- расчет контрольной суммы;
- формирование таблицы источников для UI;
- формирование Markdown-описания источников.

## 6. Критерии приемки

Функция считается реализованной, если:

- PDF-источник копируется внутрь проекта;
- исходный локальный путь не используется как основной путь;
- реестр источников создается автоматически;
- отсутствующий PDF определяется как ошибка;
- локальный путь определяется как предупреждение;
- тесты покрывают добавление, валидацию, таблицу и Markdown-вывод.

## Phase II — B.8 Documentation Evidence & Citation Audit

Добавлен слой проверки доказательной базы документации. Модуль `projects/documentation_evidence.py` проверяет реестр `docs/sources/source_registry.json`, наличие зарегистрированных PDF-файлов, ссылки вида `docs/sources/*.pdf` в документации, локальные абсолютные пути пользователя, незарегистрированные ссылки и лишние PDF-файлы без записи в реестре. Результаты доступны в виде UI-ready таблиц, manifest-структуры и Markdown-отчета для preflight/release-проверок.
